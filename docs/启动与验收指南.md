# 启动与验收指南

## 1. 环境要求

| 组件 | 版本 |
|------|------|
| JDK | 17 |
| Maven | 3.8+ |
| MySQL | 8.x |
| Node.js | 16 / 18 |
| 浏览器 | Chrome / Edge 最新版 |

## 2. 数据库初始化

```sql
CREATE DATABASE lab_system DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE lab_system;
SOURCE C:/Users/12980/Desktop/lab-system/sql/lab_system.sql;
```

执行后会创建 13 张表 + 默认账号 4 个 + 实验室/设备/预约示例数据。

## 3. 后端启动

修改 `lab-backend/src/main/resources/application.yml` 的 `spring.datasource.username/password` 为你本机 MySQL 账号。

```bash
cd C:\Users\12980\Desktop\lab-system\lab-backend
mvn clean package -DskipTests
mvn spring-boot:run
```

控制台出现 `Started LabBackendApplication` 后表示启动成功，监听 `http://localhost:8080/api`。

可用 curl 自测：
```bash
curl -X POST http://localhost:8080/api/auth/login -H "Content-Type: application/json" -d "{\"username\":\"admin\",\"password\":\"123456\"}"
```
返回 `{"code":200,...,"data":{"token":"...."}}` 即正常。

## 4. 前端启动

```bash
cd C:\Users\12980\Desktop\lab-system\lab-frontend
npm install            # 第一次安装依赖
npm run serve          # 启动开发服务器
```

浏览器自动打开 `http://localhost:8081`，输入 `admin / 123456` 登录。

## 6. 模块互通关系（评审重点）

- **用户/角色/菜单 → 所有模块**：所有 API 通过 RBAC 拦截，控制台按钮按 `perm` 隐藏。
- **实验室 / 设备 → 预约**：预约时必选实验室；同实验室同时段无法重复预约（后端 `lab_reservation.lab_id + start_time + end_time` 联合校验）。
- **设备 → 维修单**：报修自动把设备状态切到"维修"；完工自动恢复"在用"。
- **预约 → 耗材出入库**：表 `stock_record.reservation_id` 已预留，模块4 实现时直接联动。
- **全部数据 → 统计报表**：模块5 占位的 `/api/stat/usage` 后续仅需 SQL 聚合查询。

## 7. 端口与代理

## 9. 常见问题

1. **启动报 BCrypt 解码失败** —— 密文必须以 `$2a$10$` 开头；如果你重新插入用户记得用 BCrypt 加密。
2. **跨域** —— 已在 `CorsConfig` 全局放开。
3. **JWT 过期** —— 默认 12 小时，过期会触发前端弹框跳登录。
4. **MySQL 5.7** —— 也兼容；只需保证字符集 utf8mb4。
